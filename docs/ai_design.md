# AI 能力设计

## 1. 输入层：自然语言输入

用户可以输入接近日常表达的提醒句子，例如：

- 明天早上九点提醒我交作业
- 今晚八点提醒我背单词
- 每天下午三点提醒我喝水

输入层的目标是降低结构化表单的填写成本，让用户先表达意图，再由系统辅助拆解。

## 2. 理解层：解析事件、时间、重复周期

MVP 使用本地规则解析，识别常见中文时间词、重复词和任务内容：

- 时间词：今天、明天、后天、今晚、早上、下午、晚上。
- 小时表达：九点、八点半、15:30。
- 重复表达：每天、每周、每星期。
- 任务内容：去除“提醒我”等触发词后保留事项。

未来接入 LLM 后，可以让模型输出稳定 JSON，并处理更复杂的语义，例如“下周一早会前半小时提醒我准备材料”。

## 3. 决策层：是否需要用户确认

系统根据解析完整度判断是否需要确认：

- 能解析出任务和时间：直接填入表单，并展示置信度。
- 无法解析时间：提示用户手动确认或修改。
- 时间已经过去：自动顺延到下一天，或提示用户检查。

决策层的目标不是完全替代用户，而是在不确定时把控制权交还给用户。

## 4. 执行层：创建本地提醒

确认后，App 会：

1. 校验标题、内容、日期和时间。
2. 请求手机通知权限。
3. 调用 expo-notifications 创建本地通知。
4. 把提醒数据保存到 AsyncStorage。
5. 在提醒列表中展示状态。

## 5. 反馈层：提醒完成后记录状态

用户可以标记完成或删除提醒。完成状态会回写本地存储，帮助用户形成任务闭环。未来可以统计完成率、延迟率和高频提醒类型。

## 6. 未来可扩展为 Agent 的方向

- 主动询问：当用户输入“明天提醒我开会”但没有时间时，Agent 主动追问具体时间。
- 智能建议：根据历史习惯建议提醒时间。
- 多轮创建：用户可以连续补充“改成每周三下午”。
- 场景联动：接入日历、课程表、邮件或待办系统。
- 复盘总结：每天晚上总结已完成和未完成提醒。
